# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

A single Claude Code skill: `pre-launch-checklist`. It runs a 25-point interactive production readiness audit before shipping any web application.

## Skills

| File | Purpose |
|------|---------|
| `pre-launch-checklist.skill` | 25-point ops/infra readiness audit (load, database, async, resilience, observability, operations) |
| `vibe-coder-security-checklist.skill` | 30-point security audit (auth, API security, database, infrastructure, code hygiene) |
| `SKILL.md` | `pre-launch-checklist` packaged as a zip archive for distribution — keep in sync with the `.skill` file |

The two skills are intentionally separate: the pre-launch checklist covers infrastructure failure modes; the security checklist covers application-layer vulnerabilities. They have overlapping items (rate limiting, secrets, backups) that are scoped differently — do not merge them.

## Skill Structure

The `.skill` file uses this format:
```
---
name: <skill-name>
description: "<natural language trigger phrases and activation conditions>"
---

# <Skill Title>

<Markdown instructions for Claude>
```

The `description` field drives when the skill auto-activates. It must include all trigger phrases listed in `README.md`.

## Severity Classification

Items 1, 2, 3, 6, 9, 13, 14, 15, 19, 21, 25 are **blockers** (must fix before launch). All others are **warnings** (fix within 30 days). Never change this without updating both the skill file and `README.md`.

## Editing Rules

- Any change to checklist items must be reflected in: `pre-launch-checklist.skill`, `SKILL.md` (inside the zip), and the summary table in `README.md`.
- The `SKILL.md` file is a binary zip archive — do not edit it as text. Regenerate it by zipping `pre-launch-checklist/SKILL.md` if changes are needed.
- The output format section in the skill must always produce the three-bucket report: BLOCKERS / WARNINGS / PASSED.
